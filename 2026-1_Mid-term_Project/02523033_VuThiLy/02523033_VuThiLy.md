# 🔍 Research Gap Identification Assistant

> 💬 *A multi-agent AI system that helps researchers find what's missing in the literature — structured, transparent, and human-in-the-loop.*

---

# 🧩 Part 1 — The Problem

> **"Researchers spend weeks reading papers, yet still can't answer one question: What hasn't been done yet?"**

Literature review is harder than it looks:

| Pain Point                  | What Actually Happens                                  |
|-----------------------------|--------------------------------------------------------|
| Information overload        | Read many papers — retain fragmented insights          |
| No cross-paper thinking     | Each paper read in isolation — connections missed      |
| AI tools agree too much     | Most AI summarizes — it doesn't challenge              |
| No clear direction          | After reading, still don't know where to go next       |

> ⚠️ **The result?** Time wasted on a direction that already exists — or one with a flaw nobody caught early enough.

---

# 👥 Part 2 — Who Is This For?

> *"I just read 5 papers. I know what they say. But I still don't know what to research next."*

This tool is built for **one specific user** with **one specific recurring pain:**

- **Graduate students** (Master's / PhD) doing literature review
- **Researchers** scoping a new project
- **R&D engineers** identifying innovation opportunities

> 💡 This task **repeats every time a new research direction begins** — and no existing tool (ChatGPT, Google Scholar, Semantic Scholar) does it in a structured, traceable, multi-perspective way.

---

# 💡 Part 3 — The Core Idea

> **Transform a set of paper abstracts into a structured map of what's known, what's missing, and what to do next.**

### Input — What the user provides

Each paper is entered as a plain text block:

    Title:    [paper title]
    Authors:  [author names]
    Year:     [publication year]
    Abstract: [150–300 words]

- **2–5 papers** per session — pasted directly, no file upload
- **Year field** enables chronological reasoning — the system tracks how the field evolved over time

### Output — What the system returns

| #  | Component              | What It Tells You                              |
|----|------------------------|------------------------------------------------|
| 1  | Paper Summaries        | Core method + key finding per paper            |
| 2  | Limitations            | What each paper cannot do                      |
| 3  | Cross-Paper Comparison | Agreements, conflicts, evolution over time     |
| 4  | Research Gaps          | What no paper has addressed                    |
| 5  | Future Directions      | Concrete next steps                            |
| 6  | Confidence Score       | Low / Medium / High + explanation              |

---

# ⚙️ Part 4 — System Architecture

> **1 Gemini model. 4 distinct prompts. 1 sequential pipeline.**

    ┌──────────────────────────────────────────┐
    │              USER INPUT                  │
    │   2–5 Abstracts + Metadata (plain text)  │
    └─────────────────┬────────────────────────┘
                      │
                      ▼
         ┌────────────────────────┐
         │     Paper Analyzer     │  ← Summarize · Extract limitations
         └────────────┬───────────┘
                      │
                      ▼
         ┌────────────────────────┐
         │      Comparator        │  ← Compare · Track evolution by Year
         └────────────┬───────────┘
                      │
                      ▼
         ┌────────────────────────┐
         │     Gap Generator      │  ← Identify gaps · Suggest directions
         └────────────┬───────────┘
                      │
                      ▼
         ┌────────────────────────┐
         │       Verifier         │  ← Check consistency · Assign confidence
         └────────────┬───────────┘
                      │
                      ▼
    ┌──────────────────────────────────────────┐
    │            STRUCTURED OUTPUT             │
    │  Summaries · Gaps · Directions · Score   │
    └──────────────────────────────────────────┘

| Role           | What Gemini Is Prompted To Do                                   |
|----------------|-----------------------------------------------------------------|
| Paper Analyzer | Summarize each paper, extract stated + implied limitations      |
| Comparator     | Compare side by side, track chronological evolution             |
| Gap Generator  | Surface what's missing, suggest concrete future directions      |
| Verifier       | Ensure gaps trace to real limitations, assign confidence        |

> 🔑 Not four separate models — one Gemini call per role, each with a purpose-built prompt. *Separation of concerns through prompting, not tooling.*

---

# 🔄 Part 5 — How It Works

    Step 1 → Paste 2–5 abstracts with metadata
    Step 2 → Click "Analyze"
    Step 3 → Review structured output
    Step 4 → Regenerate if needed
    Step 5 → Refine input and retry

> 👤 **The user is always in control** — they choose the papers, evaluate the gaps, and decide the direction. The AI structures the thinking. It never makes the final call.

---

# 🤝 Part 6 — Human-in-the-Loop Design

> **AI surfaces insights. Humans make decisions.**

| Step                                  | Owner  |
|---------------------------------------|--------|
| Select which papers to analyze        | Human  |
| Summarize and extract limitations     | AI     |
| Compare papers and identify gaps      | AI     |
| Validate whether gaps are real        | Human  |
| Decide which direction to pursue      | Human  |

---

# 🛡️ Part 7 — Verification & Trust

| Design Choice                              | Why It Matters                                  |
|--------------------------------------------|-------------------------------------------------|
| Every gap traces back to a limitation      | Reasoning chain is fully visible                |
| Confidence score has written explanation   | Shows why, not just a label                     |
| Facts separated from inference             | Clear what came from paper vs. AI               |
| Regenerate always available                | No single output is ever treated as final       |

# 📊 Part 8 — Demo Scenario

### Input

    Title:    Transformer-Based Clinical NLP
    Authors:  Smith et al.  |  Year: 2021
    Abstract: Transformer model for clinical records. Evaluated on
              3 English benchmarks. State-of-the-art results.
              Limitation: English-language data only.

    Title:    CNN for Medical Image Classification
    Authors:  Lee et al.  |  Year: 2023
    Abstract: CNN for classifying medical images. 94% accuracy.
              Limitation: requires large labeled datasets, does not
              generalize across hospital systems.

### Output

**Summaries**
- (2021) Smith et al. → Clinical NLP transformer | English only
- (2023) Lee et al.   → Medical image CNN        | Data-hungry, low generalization

**Comparison**
- Separate modalities — text (2021) vs. image (2023) — no integration attempted
- Both depend on large single-source datasets

**Research Gaps**
1. No joint model integrating clinical text + medical images
2. No multilingual clinical NLP across hospital systems
3. No low-resource / few-shot approach for real clinical deployment

**Suggested Directions**
1. Multimodal model combining NLP + vision for clinical AI
2. Cross-lingual transfer learning for multilingual records
3. Few-shot framework for data-scarce hospital environments

**Confidence: Medium**
> Multimodal gap strongly supported. Multilingual gap needs more papers to confirm.

# 🛠️ Part 9 — Tech Stack & Scope

| Layer         | Technology                  | Role                                   |
|---------------|-----------------------------|----------------------------------------|
| UI            | Google Antigravity (Python) | Input form + output display            |
| All Roles     | Google Gemini API           | All reasoning — 4 calls, 4 prompts     |
| Orchestration | Python (sequential)         | Pass output role → role                |
| Config        | .env + python-dotenv        | Secure API key management              |

**In Scope**
- Structured text input (title + authors + year + abstract)
- 4-role sequential pipeline — one Gemini model, four purpose-built prompts
- Chronological reasoning via Year field
- All 6 output components + regenerate + confidence score
- Clean UI built in Google Antigravity

**Out of Scope**
- PDF upload / parsing · Real-time web search · Persistent storage · Parallel calls

# 🎯 Part 10 — Design Principles & Impact

    Insight over summary     — reveal what's missing, not just what's there
    Trace every claim        — gaps must come from real limitations
    Human in the loop        — AI advises, human decides
    Chronological awareness  — year matters, field evolution matters
    Honest uncertainty       — confidence score is not decoration

| Before                              | After                                      |
|-------------------------------------|--------------------------------------------|
| Days reading, no clear direction    | 30 min → structured gap map                |
| Missed cross-paper connections      | Explicit chronological comparison          |
| Vague research questions            | Concrete, traceable gap statements         |
| AI that agrees with everything      | Structured critique + confidence score     |
