# Project Goal: PDF OCR & AI Analysis Pipeline

## Purpose

Scanned PDFs (image-only, no text layer) cannot be directly fed into AI models for analysis.
This project automates the full pipeline — OCR → AI analysis → structured output — to
eliminate the manual effort of processing large batches of image-based PDFs.

---

## ✅ Already Completed

- **Streamlit GUI app** — fully working web interface
- **PDF text extraction** — PyMuPDF-based native text layer extraction
- **AI analysis** — extracted text sent to LLM, results returned
- **Chart generation** — visual output from analyzed data, displayed in Streamlit
- **Multi-API support** — Gemini, OpenAI, Ollama (local) all integrated

---

## 🔲 Planned Features

### 1. Image → OCR
- Render each PDF page as a high-resolution image
- Send page images to OCR API (Gemini / OpenAI / Ollama vision model)
- Extract text with bounding box positions per block

### 2. AI Analysis → Multiple `.md` Files
- Analyze extracted text per page or per batch
- Output structured Markdown files (one per document or per section)
- **API usage throttle**: auto or manual control over how many pages are sent per API call,
  based on task complexity and difficulty — adjustable by the user

### 3. Figure Extraction
- Detect and crop non-text regions (diagrams, charts, photos) using AI
- Save each figure as a separate image file (`figures/pageN_figM.png`)

### 4. PDF Reconstruction
- Reassemble a clean, searchable PDF from extracted text + figures
- Place text blocks and figures at their original detected positions

### 5. Summarization & Q&A
- Generate a summary of the full document
- Allow the user to ask questions about the document content
- Answers returned in the Streamlit interface

---

## Target Workflow (per job)

```
User sets: input PDF(s), API backend, pages-per-call

 PDF pages
    │
    ▼
[OCR — Image → Text + Figure regions]
    │
    ▼
[AI Analysis — Text → Structured .md files]   ← API usage control applied here
    │
    ├──▶ figures/  (cropped image files)
    ├──▶ output.md (per document)
    ├──▶ output_converted.pdf (reconstructed)
    ├──▶ Summary panel (Streamlit)
    └──▶ Q&A chat (Streamlit)
```
